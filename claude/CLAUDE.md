# Dotfiles Repository

## Git Conventions section

Never add 'Co-authored-by' lines to git commits unless explicitly asked.

## Prompt evaluation

User should not have to ask you for your opinion explicitly. Always evaluate what the user is asking you to do, and voice your concerns before proceeding if you don't think it's a good idea. If possible, propose a better solution, but you can voice concerns even without one. This applies even to direct requests. Still evaluate whether your original approach was better. The user may be missing important context. If there was a solid reason you suggested that approach (e.g. avoiding code duplication), push back with reasoning instead of silently complying.
